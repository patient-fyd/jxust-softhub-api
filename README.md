# jxust-softhub-api

江西应用科技学院软件协会官网 & 后台管理系统后端项目

------

## 📌 项目介绍

`jxust-softhub-api` 是本系统后端接口服务，基于 GoFrame2 + MySQL/PostgreSQL 开发，负责为 **官网前端（jxust-softhub-web）** 和 **后台管理系统（jxust-softhub-admin）** 提供 RESTful API，涵盖用户、权限、新闻、活动、项目、资源、审核、统计、互动等核心模块。

同时支持与已有的 **C++ OJ 系统** 和 **搜索引擎系统** 集成，保证系统数据统一、接口高可用、安全可靠。

------

## 🛠 技术栈

- **开发语言**：Go v1.18+
- **框架**：GoFrame v2.x
- **数据库**：MySQL / PostgreSQL
- **缓存**：Redis
- **鉴权**：JWT + RBAC 权限模型
- **ORM**：GoFrame ORM
- **API风格**：RESTful
- **日志**：GoFrame 自带 g.Log + 业务审计日志

------

## ✅ 功能模块

### 一、用户与权限系统

- 用户注册 / 登录（JWT）
- 用户信息管理
- 用户角色管理（超级管理员、内容管理员、普通成员）
- 权限控制（基于RBAC）
- 前后台用户分离登录

------

### 二、新闻动态

- 新闻分类管理（协会新闻、技术分享、赛事通知）
- 新闻内容管理（增删改查）
- 新闻详情接口（支持 Markdown 渲染）
- 新闻搜索（接入搜索引擎接口）

------

### 三、活动管理

- 活动信息管理（增删改查）
- 活动详情
- 报名接口（前台成员）
- 报名数据管理（后台）
- 报名审核（通过 / 驳回 / 导出）
- 报名状态（未开始 / 进行中 / 已结束）

------

### 四、项目管理

- 项目基本信息管理
- OJ、搜索引擎系统集成（超链接或内嵌）
- 其他自主项目展示接口

------

### 五、成员与加入管理

- 成员信息展示
- 核心成员管理
- 协会成员分届管理
- 入会申请接口（前台）
- 入会申请审核接口（后台）
- 入会状态查询

------

### 六、资源管理

- 资源分类管理
- 文件上传 / 下载接口
- 视频、文档、图片、压缩包支持

------

### 七、评论 & 消息系统

- 新闻 & 活动评论
- 用户私信
- 通知中心
- 内容审核机制

------

### 八、统计分析

- 网站访问统计（PV/UV）
- 活动报名数据统计
- 用户活跃度、评论、私信等交互统计
- 数据接口供后台 ECharts 可视化

------

### 九、日志系统

- 操作日志
- 审计日志
- 关键操作报警（可拓展）

------

## ✅ 服务端接口分类

| 模块              | 对应前端        | 说明                           |
| ----------------- | --------------- | ------------------------------ |
| AuthService       | 官网 + 管理后台 | 登录 / 注册 / Token 校验       |
| UserService       | 管理后台        | 用户、角色、权限管理           |
| NewsService       | 官网 + 管理后台 | 新闻管理 + 新闻展示接口        |
| ActivityService   | 官网 + 管理后台 | 活动发布、报名、统计           |
| MemberService     | 官网            | 成员展示、分届筛选             |
| ProjectService    | 官网            | 项目列表、OJ集成、搜索引擎集成 |
| ResourceService   | 官网 + 管理后台 | 资料上传、下载、分类管理       |
| CommentService    | 官网            | 评论、私信、通知中心           |
| StatisticsService | 管理后台        | 数据分析与接口输出             |
| AuditService      | 管理后台        | 审核、日志记录、操作留痕       |

## ✅ commit 规范

| 类型     | 说明                     |
| -------- | ------------------------ |
| feat     | 新增接口 / 功能          |
| fix      | 修复 Bug                 |
| refactor | 重构代码，不影响接口     |
| style    | 代码风格 / 空格 / 格式化 |
| chore    | 构建、脚本、CI/CD        |
| docs     | 文档更新                 |
| test     | 单元测试                 |

### commit 示例

```bash
feat: 完成用户注册接口
fix: 修复 JWT Token 刷新异常
refactor: 重构报名统计接口
docs: 添加活动报名接口文档
```

## 🚩 开发 & 运行

### 1. 依赖安装

```bash
go mod tidy
```

### 2. 配置环境

- 配置文件：`config.yaml`（数据库、Redis、JWT、静态文件、日志等）
- 环境变量：可选使用 `.env` 进行环境参数管理

### 3. 启动项目

```bash
go run main.go
```

### 4. 编译

```bash
go build -o bin/jxust-softhub-api main.go
```

------

## 🏷 注意事项

- 所有接口遵循 RESTful 规范
- 统一返回格式：`code + msg + data`
- 前端严格校验 Token 及权限
- 后端所有接口都应支持 JWT 鉴权 & 权限校验
- 管理端接口权限应严格按角色控制
- 与 OJ、搜索引擎系统通过 HTTP API 对接

------

✅ 推荐：

- 使用 Swagger 或 Apifox 维护接口文档
- 使用 GoFrame2 内置中间件（权限、限流、日志）+ 自定义扩展
- 接口开发时保持前后端接口风格统一
